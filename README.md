# Веб-сервис для тестирования пользовательских функциональных модулей

Описание:
Веб-сервис предназначен для загрузки, тестирования и использования пользовательских Python-модулей различного уровня сложности, включая алгоритмы, ИИ-модели и утилиты.
Сервис позволяет создавать модули с определённой структурой, привязывать их к ролям пользователей и динамически формировать интерфейс для работы с ними.

---

Основные возможности:

1. Загрузка модулей и тестов
   - Поддержка загрузки готовых `.py` файлов с модулями, соответствующими заданной структуре.
   - Возможность загрузки unit-тестов в формате JSON для проверки корректности работы модулей.

2. Управление ролями и доступом
   - Модули можно прикреплять к одной или нескольким ролям без ограничений.
   - Пользователь может иметь несколько ролей, каждая из которых даёт доступ к определённым модулям.

3. Использование ChatGPT
   - Пользователи могут использовать встроенный ChatGPT в рамках своего профиля.
   - Для работы с ChatGPT требуется API-токен.

4. Динамический интерфейс модулей
   - Для каждого модуля создаётся уникальный интерфейс на основе метаданных.
   - Поддерживаются текстовые поля, поля для загрузки файлов и другие элементы ввода.
   - Интерфейс формируется автоматически в зависимости от настроек разработчика модуля.

---

Начало работы:

1. Склонируйте проект с GitHub
   git clone https://github.com/6oT9lpa/website-for-testing-custom-functional-modules.git

2. Создайте виртуальное окружение и установите зависимости
   python -m venv venv
   venv/Scripts/activate   # Windows
   source venv/bin/activate # Linux / macOS
   pip install -r requirements.txt

3. Запустите проект
   python run.py

4. Откройте веб-сервис
   Перейдите на http://127.0.0.1:5000 и создайте аккаунт.
   Для доступа к админ-панели (если у пользователя нет роли администратора) используйте:
   http://127.0.0.1:5000/admin-panel

5. Создание нового модуля
   - Перейдите во вкладку Модули → Создать модуль.
   - Заполните необходимые поля и сохраните модуль.
   - Редактировать модуль можно через встроенный редактор кода.

6. Присвоение модуля пользователю
   - Прикрепите созданный модуль к одной или нескольким ролям.
   - Добавьте роль пользователю, которому необходим доступ к модулю.

---

Важно:

- Защита админ-панели и профиля пока отсутствует, используйте сервис только в безопасной среде.
- Для работы ChatGPT необходим действующий API-токен.

---

Краткое описание функционала проекта:

- Разработана система с строгой структурой модулей (функция запуска, описание полей для тестирования).
- Реализован динамический UI для каждого модуля на основе метаданных.
- Интегрирован функционал для загрузки unit-тестов и редактирования кода через веб-интерфейс.
